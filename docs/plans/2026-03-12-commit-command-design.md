# Команда /commit Design

## Контекст

В репозитории нет локальной slash-команды Claude Code для создания качественных git-коммитов. Из-за этого история коммитов деградирует в шумовые сообщения без смысла и без связи с реальным масштабом изменений.

## Цель

Добавить локальную команду `/commit`, которую можно вызывать в Claude Code для автоматического сохранения текущих изменений с осмысленным сообщением коммита на русском языке.

## Требования

- Команда должна сама сохранять изменения через git.
- Сообщение коммита должно быть на русском языке.
- Формулировка должна быть в прошедшем времени: `добавил`, `исправил`, `перенёс`, `переработал`.
- Инфинитивы вроде `добавить` и `исправить` запрещены.
- При локальных изменениях команда должна писать короткий заголовок.
- При крупных изменениях команда должна писать заголовок и развёрнутое тело коммита.
- Команда не должна подменять автора через `--author`.

## Подход

Команда будет реализована как локальный markdown-файл в `.claude/commands/commit.md`. Claude Code уже использует такой формат для slash-команд, поэтому отдельные скрипты и обвязки не нужны.

Команда будет:

1. Читать текущий git status, ветку, diff и последние коммиты.
2. Индексировать все текущие изменения через `git add -A`.
3. Оценивать масштаб изменений по количеству файлов и по смысловым областям diff.
4. Выбирать один из двух форматов:
   - короткий заголовок для узких изменений;
   - заголовок и тело для широких изменений.
5. Выполнять `git commit` без передачи `--author`.

## Формат сообщения

Заголовок:

`<type>: <действие в прошедшем времени>`

Примеры:

- `fix: покрасил кнопку входа`
- `feat: добавил sitemap и robots`
- `refactor: вынес проверку доступа к маршрутам в отдельный модуль`

Для крупных изменений добавляется тело коммита с кратким перечислением смысловых блоков изменений на русском языке.

## Ограничения

- Если изменений нет, команда не должна создавать пустой коммит.
- Если после `git add -A` нечего коммитить, команда должна остановиться без побочных действий.
- Команда должна избегать размытых формулировок вроде `обновил код` или `внёс изменения`, если diff позволяет сказать точнее.
