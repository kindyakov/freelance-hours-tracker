# Commit Command Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Добавить локальную команду `/commit` для Claude Code, которая автоматически индексирует изменения и создаёт осмысленный git-коммит на русском языке.

**Architecture:** Команда реализуется как markdown-файл в `.claude/commands`, где Claude Code получает контекст из git-команд и следует жёстким правилам генерации commit message. Логика остаётся декларативной: без внешних скриптов и без дублирования поведения git.

**Tech Stack:** Claude Code slash commands, Git, Markdown frontmatter

---

### Task 1: Подготовить документацию решения

**Files:**
- Create: `docs/plans/2026-03-12-commit-command-design.md`
- Create: `docs/plans/2026-03-12-commit-command.md`

**Step 1: Зафиксировать цель и ограничения**

Описать, зачем нужна команда `/commit`, какие требования к русскому тексту коммита обязательны и почему подмена автора запрещена.

**Step 2: Зафиксировать формат поведения**

Описать два режима работы: короткий коммит для локальных изменений и заголовок с телом для крупных изменений.

**Step 3: Проверить, что документация отражает согласованный дизайн**

Убедиться, что в документации явно запрещены инфинитивы и расплывчатые формулировки.

### Task 2: Добавить локальную slash-команду

**Files:**
- Create: `.claude/commands/commit.md`

**Step 1: Описать поддерживаемые git-инструменты**

Добавить frontmatter с `allowed-tools` для `git status`, `git diff`, `git add`, `git commit`, `git log` и `git branch`.

**Step 2: Передать команде нужный контекст**

Вставить в markdown команды для получения текущего статуса, ветки, diff до индексации и недавней истории коммитов.

**Step 3: Сформулировать жёсткие правила**

Явно потребовать:
- индексировать все изменения через `git add -A`;
- писать сообщение коммита на русском языке в прошедшем времени;
- использовать короткий заголовок для узких изменений;
- использовать заголовок и тело для крупных изменений;
- не использовать `--author`.

**Step 4: Ограничить поведение**

Запретить любые действия кроме индексации и коммита. Если коммит невозможен, команда должна остановиться без побочных действий.

### Task 3: Проверить результат

**Files:**
- Modify: `.claude/commands/commit.md`

**Step 1: Прочитать итоговый markdown**

Проверить, что команда читается однозначно и не оставляет места для мусорных commit message.

**Step 2: Проверить структуру проекта**

Убедиться, что `.claude/commands` появился в репозитории и команда будет локальной для этого проекта.

**Step 3: Подготовить краткую инструкцию по использованию**

Сформулировать, что достаточно вызвать `/commit`, после чего агент сам проанализирует изменения и выполнит git commit.
