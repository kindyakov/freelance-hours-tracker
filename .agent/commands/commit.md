## Контекст

- Текущий git status: !`git status --short`
- Текущая ветка: !`git branch --show-current`
- Последние коммиты: !`git log --oneline -10`
- Текущий diff до индексации: !`git diff HEAD`

## Задача

Создай один git-коммит по текущим изменениям в рабочем дереве.

Правила обязательны:

1. Сначала проиндексируй все текущие изменения через `git add -A`.
2. После индексации создай ровно один коммит.
3. Не используй `--author` и не подменяй автора никаким другим способом.
4. Пиши сообщение коммита только на русском языке.
5. Формулируй действие только в прошедшем времени: `добавил`, `исправил`, `перенёс`, `переработал`, `покрасил`.
6. Инфинитивы запрещены: нельзя писать `добавить`, `исправить`, `обновить`.
7. Если изменения узкие и касаются одной области, используй только короткий заголовок.
8. Если изменения крупные, затрагивают несколько областей или меняют поведение системы заметно, используй заголовок и развёрнутое тело коммита.
9. Заголовок должен быть конкретным. Избегай мусорных фраз вроде `обновил код`, `внёс изменения`, `правки`.
10. Используй подходящий conventional type, если он уместен: `feat`, `fix`, `refactor`, `docs`, `test`, `chore`.

Формат:

- Для локальных изменений: `<type>: <короткое действие в прошедшем времени>`
- Для крупных изменений:
  - первая строка: `<type>: <обобщающее действие в прошедшем времени>`
  - затем пустая строка
  - затем 2-4 короткие строки тела коммита, каждая строка должна конкретизировать отдельный блок изменений на русском языке

Примеры хороших заголовков:

- `fix: покрасил кнопку входа`
- `feat: добавил sitemap и robots`
- `refactor: вынес проверку доступа к маршрутам в отдельный модуль`

Если после `git add -A` коммитить нечего, не создавай пустой коммит.

Используй только tool calls для `git add` и `git commit`. Не пиши никакого дополнительного текста.
