# RoastFlow

Простой трекер рабочего времени и заработка для фрилансера.  
Без перегруженных CRM-подобных интерфейсов, лишних сущностей и десятков вкладок.

Репозиторий: [github.com/kindyakov/freelance-hours-tracker](https://github.com/kindyakov/freelance-hours-tracker)

## Зачем этот проект

Большинство time tracker-сервисов пытаются быть сразу всем: учётом проектов, команд, биллингом, аналитикой, автоматизациями и ещё кучей настроек. В итоге простая задача "записать часы и понять, сколько заработал" превращается в работу с тяжёлой системой.

RoastFlow решает только одну задачу:

- быстро записать рабочие интервалы;
- зафиксировать заработок по датам;
- увидеть месяц целиком без лишнего шума.

## Для кого

- для фрилансера, которому нужен личный рабочий журнал;
- для тех, кто хочет учитывать и часы, и деньги в одном месте;
- для тех, кому важна простота, а не бесконечная кастомизация.

Если нужен командный тайм-трекинг, invoicing-платформа или ERP-комбайн, это не тот продукт.

## Что умеет

- **Учёт часов**: дневные записи с временными сегментами и категориями активности.
- **Учёт заработка**: несколько поступлений в месяц с конкретными датами.
- **Дашборд**: часы, категории, статистика и заработок за текущий месяц.
- **История**: помесячный обзор накопленных данных.
- **Авторизация**: вход через GitHub OAuth.

## Как выглядит сценарий использования

1. Добавили день и рабочие интервалы.
2. Внесли доход по фактическим датам.
3. Открыли дашборд или историю и сразу увидели картину по месяцу.

## Стек

| Слой         | Технология                 |
| ------------ | -------------------------- |
| Framework    | Next.js 16 (App Router)    |
| UI           | Mantine v8 + Tailwind v4   |
| Database     | PostgreSQL + Prisma 7      |
| Auth         | NextAuth v5 (GitHub OAuth) |
| Client State | Zustand + TanStack Query   |
| Deploy       | Vercel                     |

## Локальный запуск

```bash
npm install
```

Создайте `.env.local` вручную и заполните:

```env
DATABASE_URL=
DIRECT_URL=
AUTH_SECRET=
AUTH_GITHUB_ID=
AUTH_GITHUB_SECRET=
NEXTAUTH_URL=http://localhost:3000
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

Примените миграции и запустите приложение:

```bash
npx prisma migrate deploy
npx prisma generate
npm run dev
```

После запуска откройте [http://localhost:3000](http://localhost:3000).

## Команды

```bash
npm run dev         # локальная разработка
npm run build       # production build
npm run start       # запуск production build
npm run lint        # eslint
npx tsc --noEmit    # проверка типов
npx prisma studio   # просмотр базы данных
```

## Переменные окружения

| Переменная             | Назначение                                          |
| ---------------------- | --------------------------------------------------- |
| `DATABASE_URL`         | Основное подключение к базе                         |
| `DIRECT_URL`           | Прямое подключение для миграций                     |
| `AUTH_SECRET`          | Секрет NextAuth                                     |
| `AUTH_GITHUB_ID`       | GitHub OAuth Client ID                              |
| `AUTH_GITHUB_SECRET`   | GitHub OAuth Client Secret                          |
| `NEXTAUTH_URL`         | Базовый URL приложения                              |
| `NEXT_PUBLIC_SITE_URL` | Публичный URL для canonical, sitemap и SEO metadata |

## Философия

Проект намеренно не разрастается в "сервис для всего". Его ценность именно в ограниченности: чем меньше трение при записи часов и дохода, тем выше шанс, что инструментом будут реально пользоваться каждый день.
